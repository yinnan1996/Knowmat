# KnowMat: An AI-Guided Material Design Agent with Knowledge-Integrated Tool Learning Mechanisms

KnowMat is a material design reasoning system that uses **dependency-aware dynamic planning** to guide multi-step workflows. The core algorithm constructs explicit dependency graphs (DAGs) to ensure task decomposition, tool invocation, and intermediate data flow follow clear and verifiable dependencies.

---

## Core Algorithm: Dependency-aware Dynamic Planning

### Overview

Multi-step material design tasks are formalized as a **directed acyclic graph (DAG)**:

$$G = (V, E)$$

- Each node $v_i \in V$ corresponds to a subtask $t_i$ generated by the LLM
- Each edge $(v_j \rightarrow v_i) \in E$ indicates that task $t_i$ depends on the output of task $t_j$

Each subtask is a quadruple:

$$t_i = (\mathrm{id}_i,\, \mathrm{dep}_i,\, \mathrm{tool}_i,\, \mathrm{args}_i)$$

- $\mathrm{dep}_i$: indices of upstream tasks ($\mathrm{dep}_i = \{-1\}$ for root tasks)
- $\mathrm{tool}_i$: tool from the available tool set $\mathcal{T}$

### DAG Generation

A structured prompt $\mathcal{P}(q, \mathcal{T})$ includes the user query, tool descriptions with I/O specs, and few-shot examples. The LLM generates a JSON task list; the DAG is built by adding edges $(v_j \rightarrow v_i)$ for each $j \in \mathrm{dep}_i$, $j \neq -1$.

### Parameter Substitution & Topological Execution

Placeholders `<GENERATED>-d` in arguments are replaced by the output of task $t_d$:

$$\tilde{a}_{ik} = \mathrm{Substitute}(a_{ik}, R) = a_{ik}\big|_{\langle \mathrm{GENERATED} \rangle\text{-}d \mapsto R[t_d]}$$

Execution follows a **topological order**: nodes with zero in-degree run first; upon completion, successor in-degrees are reduced. Complexity: $\mathcal{O}(|V|+|E|)$.

### Retry & Localized Re-planning

- **Retry**: Each tool supports configurable retries; success probability $P_s = 1 - p_f^{R+1}$ after $R+1$ attempts
- **Re-planning**: On persistent failure, partial results $R$ are given to the LLM to regenerate a corrected DAG, allowing continuation without full restart

## Installation

### 1. Clone and install dependencies

```bash
cd knowmat
pip install -r requirements.txt
```

### 2. Configuration

```bash
cp backend/src/.env.example backend/src/.env
```

Edit `.env` and set:

- `LLM_API_KEY`: Your LLM API key (OpenAI-compatible, e.g. Aliyun DashScope, DeepSeek)
- `LLM_BASE_URL`: API base URL
- `LLM_MODEL_ID`: Model name (e.g. `qwen-plus-latest`)
- `DB_HOST`, `DB_PORT`, `DB_USER`, `DB_PWD`, `DB_NAME`: PostgreSQL credentials

### 3. Run the backend

```bash
cd backend/src
python chat_server.py   # or: ../../.venv/bin/python chat_server.py
```

The server listens on `http://0.0.0.0:7896` (configurable via `BACKEND_PORT`).

**Note:** The root endpoint `GET /` works immediately. The `POST /chat` endpoint requires a valid `LLM_API_KEY` in `.env`. For database-backed tools (e.g. `query_database`), configure PostgreSQL credentials. Set `MOCK_MODELS=1` to use placeholder alloy predictions without trained models.

When trained alloy models are not available, set `MOCK_MODELS=1` in `.env`. The system returns placeholder values for other property predictions.
